一、CSS 初识

写在 HTML 的head标签内，在title标签下面，用style标签包裹 > 选择器：{CSS 属性：属性值}

二、CSS 引入方式

  1. 内嵌式：CSS 写在 style 标签中
    • 虽然 style 标签虽然可以写在页面任意位置，但是通常约定写在 head 标签中
  2. 外联式：CSS 写在一个单独的.css 文件中
    • 需要通过 link 标签在网页中引入，link 标签写在 title 标签下方
    • <link rel="stylesheet" href="">
      • rel 属性全称为 relationship
      • stylsheet 意思是样式表
      • href 属性写入 CSS 文件的路径
  3. 行内式：CSS 写在标签的 HTML-style属性中 - 基础不推荐使用，之后配合 JS 使用
    引入方式 书写位置 作用范围 使用场景
    内嵌式 CSS 写在 style 标签中 当前页面 小案例
    外联式 CSS 写在单独的 css 文件中，通过 link 标签引入 多个页面 项目中
    行内式 CSS 写在标签的 style 属性中 当前标签 配合 JS 使用

三、基础选择器

  1. 标签选择器：
    • 结构：标签名{css属性名：属性值;}
    • 作用：通过标签名，找到页面中所有这类标签，设置样式
    • 注意点：
      • 标签选择器选择的是一类标签，而不是单独一个，选中所有的这类标签都生效 CSS
      • 标签选择器无论嵌套多深，都能找到对应的标签
  2. 类选择器：
    • 结构：.类名{css属性名：属性值;} class="类名"
    • 作用：通过类名，找到页面中所有带这个类名的标签，设置样式
    • 注意点：
      • 所有标签上都有 class 属性，class 属性的属性值称为类名
      • 类名可以由字母、数字、下划线、中划线组成，但不能以数字或者中划线开头
      • 一个标签可以同时拥有多个类名，类名之间以空格隔开
      • 类名可以重复，一个类选择器可以同时选中多个标签
  3. id 选择器：
    • 结构：#id属性值{css属性名:属性值;} id="id属性值"
    • 作用：通过 id 属性值，找到页面中带有这个 id 属性值的标签，设置样式
    • 注意点：
      • 所有标签上都有 id 属性
      • id 属性值类似于身份证号码，在一个页面中是唯一的， 不可重复的！
      • 一个标签上只能有一个 id 属性值
      • 一个 id 选择器只能选中一个标签
      • 研制出是为了配合 JS 使用，一对一使用！
  4. 通配符选择器：
    • 结构：* {css属性名:属性值;}
    • 作用:找到页面中所有的标签，设置样式
    • 注意点:
      • 开发中使用极少，只会在极特殊情况下才会用到
      • 在小页面中可能会用于去除标签默认的 margin 和 padding（例如页边距）

四、字体和文本样式

一、字体样式

  1. 字体大小：
    • 属性名：font-size
    • 取值：数字 + px 例如：xxx：50px
    • 注意点：
      • Chrome 浏览器默认文字大小是 16px
      • 单位需要设置，否则无效
  2. 字体粗细：
    • 属性名：font-weight
    • 取值：
      • 关键字：正常 normal 加粗 bold
      • 纯数字：100-900 的整百数 正常 - 400 加粗 - 700
    • 注意点：
      • 不是所有的字体都提供了九种粗细，因此部分取值页面中无变化
      • 实际开发中以正常和加粗两种取值最多
  3. 字体是否倾斜：
    • 属性名：font-style
    • 取值：
      • 正常（默认值）：normal
      • 倾斜：italic
  4. 字体系列：
    • 属性名：font-family
    • 常见取值：
      • 具体字体：“Microsoft YaHei”、微软雅黑、黑体、宋体、楷体等
      • 字体系列：sans-serif、serif、monospace 等
      • Windows 默认字体：微软雅黑
      • MacOS 默认字体：苹方
    • 渲染规则：
      • 从左到右依次查找，若电脑中未安装该字体，则显示下一字体
      • 如果都不支持，此时会根据操作系统，显示最后字体系列的默认字体
    • 注意点：
      • 如果字体名称中存在多个单词，推荐使用引号包裹
      • 最后一项字体系列不需要引号包裹
      • 网页开发时，尽量使用系统常见自带的字体，保证不同浏览网页都可以正常显示
  5. 样式的层叠问题
    • 问题：
      • 给同一个标签设置了相同的格式，此时浏览器会如何渲染？
    • 结果：
      • 如果给同一个标签设置了相同的属性，此时样式会层叠（覆盖），写在最下面的会生效
    • TIP：
      • CSS（Cascading Style Sheets）层叠样式表
      • 所谓层叠即叠加的意思，表示样式可以一层一层的层叠覆盖
.......
<style>
    p {
        color: red;
        color: blue;
    }
</style>
<body>
    <p>我是p标签</p>
</body>
....
  最终显示的是：`我是p标签`
  1. 字体 font 相关属性的连写：
    • 属性名：font(复合属性)
    • 取值：
      • font:style weight size family；
      • 例如：font:italic 700 60px 宋体
    • 省略要求：
      • 只能省略前两个，如果省略了相当于设置默认值
    • 注意点：
      • 如果需要同时设置单独和连写形式
        • 要么把单独的样式写在连写的下面
        • 要么把单独的样式写在连写的里面

二、文本样式

  1. 文本缩进：
    • 属性名：text-indent
    • 取值：
      • 数字 + 单位（px ……）
      • 特别：数字 + em （1em = 当前标签的 font-size 的大小）
  2. 文本（内容）水平对齐方式：
    • 属性名：text-align
    • 取值：
      • left：左对齐
      • center：居中对齐
      • right：右对齐
    • 注意点：
      • 如果需要让文本水平居中对齐，text-align 属性给文本所在标签（文本父元素）设置
    • 也可以用于其他内容的对齐：
      • 文本
      • span 标签，a 标签
      • input 标签，img 标签
  3. 文本修饰：
    • 属性名：text-decoration
    • 取值：
      • underline：下划线（常用）
      • line-through：删除线（不常用）
      • overline：上划线（几乎不用）
      • none：无装饰线（常用）
      • 开发中会使用text-decoration:none；清除 a 标签（链接）默认的下划线
  4. 行高：
    • 作用：控制一行的上下行间距
      • 属性名：line-height
      • 取值：
        • 数字 + 单位（px ……）
        • 倍数（当前标签 font-size 的倍数）
      • 应用：
        • 让单行文本垂直居中可以设置 line-height：文字父元素高度
        • 网页精准布局时，会设置 line-height：1 可以取消上下间距
      • 行高和 font 连写的注意点：
        • 如果同时设置了行高和 font 连写，注意覆盖问题
        • font：style weight size/line-height family

CSS 进阶

一、选择器进阶（复合选择器）：

🌾 后代选择器：空格 隔开

  - 作用：根据HTML标签的嵌套关系，选择父元素后代中满足条件的元素
  - 选择器语法：`选择器1 选择器2 { CSS }`
  - 结果：
     - 在选择器1所找到标签的后代（儿子、孙子、重孙子...）中，找到满足选择器2的标签，设置样式
  - 注意点：
     - 后代包括：儿子、孙子、重孙.....
     - 后代选择器中，选择器与选择器之前通过空格隔开

🌾 子代选择器： > 隔开

  - 作用：根据HTML标签的嵌套关系，选择父元素子代中满足条件的元素
  - 选择器语法：`选择器1>选择器2 { CSS } `
  - 结果：
     - 在选择器1所找到的子代（儿子）中，找到满足选择器2的标签，设置样式
  - 注意点：
     - 子代只包括：儿子
     - 子代选择器中，选择器与选择器之间通过 > 隔开

🌾 并集选择器：，隔开

  • 作用：同时选中多组标签，设置相同的样式
  • 选择器语法：选择器1,选择器2{ CSS }
  • 结果：
    • 找到选择器 1 和选择器 2 的标签，设置样式
  • 注意点：
    • 并集选择器中的每组选择器之间通过，分隔
    • 并集选择器中的每组选择器可以是基础选择器或者复合选择器
    • 并集选择器中的每组选择器通常一行写一个，提高代码的可读性

🌾 交集选择器：紧挨着

  • 作用：选中页面中同时满足多个选择器的标签
  • 选择器语法：选择器1选择器2{ CSS }
  • 结果：
    • （既又原则）找到页面 既 能被选择器 1 选中， 又 能被选择器 2 选中的标签，设置样式
  • 注意点：
    • 交集选择器中的选择器之间是紧挨着的，没有东西分隔
    • 交际选择器中的每组选择器可以是基础选择器或者复合选择器
    • 交集选择器中如果有标签选择器，标签选择器必须写在最前面

🌾hover 伪类选择器：

  • 作用：选中鼠标悬停在元素上的状态，设置样式
  • 选择器语法：选择器：hover{ CSS }
  • 注意点：
    • 伪类选择器选中的元素的某种状态
    • 任何一个标签都可以添加伪类，任何一个标签都可以鼠标悬停
  • 例如，鼠标在某个购物网站上一个链接悬停后字体变成红色

三、背景相关属性

🌾 背景颜色：

  • 属性名：background-color(bgc)
  • 属性值：
    • 颜色取值：关键字，RGB 表示法，RGBa 表示法，#十六进制…..
  • 注意点：
    • 背景颜色默认是透明：RGBa（0,0,0,0）、transparent
    • 背景颜色不会影响盒子大小，并且还能看清盒子的大小和位置，一般在布局中会习惯先给盒子设置背景颜色

🌾 背景图片：

  • 属性名：background-imag(bgi)
  • 属性值：background-image：url（"图片的路径"）
  • 注意点：
    • 背景图片中 url 中可以省略引号
    • 背景图片默认是在水平和垂直方向平铺的
    • 背景图片仅仅是指给盒子起到装饰效果，类似于背景颜色，是不能撑开盒子的

🌾 背景平铺：

  • 属性名：background-repeat(bgr)
  • 属性值：
    • 不平铺（no-repeat）最常用

🌾 背景图大小：

  • 作用：设置背景图片的大小
  • 语法：background-size: 宽度 高度;
  • 取值：
  • background-size 和 background 连写同时设置时，需要注意覆盖问题
    • 要么写在连写的里面，要么写在连写的下面

🌾 背景位置：

  • 属性名：background-position(bgp)
  • 属性值：background-position：水平方向位置 垂直方向位置；

  • 注意点：
    • 方位名词取值和坐标取值可以混合使用，第一个取值表示水平，第二个取值表示垂直

🌾 背景相关属性连写：

  • 属性名：background
  • 属性值：
    • 单个属性的合写，取值之间以空格隔开
  • 书写顺序：
    • 推荐：background：color image repeat position/size；
  • 省略问题：
    • 可以按需求省略
    • 特殊情况：在 PC 端，如果盒子大小和背景图大小一样，此时可以直接写 background：url()
  • 注意点：
    • 如果需要设置单独的样式：写在连写下面或里面

🌾 精灵图：

  • 场景：项目中将多张小图片合并成一个大图片，这张大图成为精灵图
  • 优点：减少服务器发送次数，减轻服务器压力，提高页面加载速度
  • 使用步骤：
    • 创建一个盒子，设置盒子的尺寸和小图尺寸相同
    • 将精灵图设置为盒子的背景图片
    • 修改背景位置
      • 通过 PxCook 测量小图片左上角图标，分别取 负值 设置给盒子的 background-position：x y
      • background-position：改变背景图的位置
      • x 取负值为向左移动，y 取负值向上移动

七、过渡

  • 作用：让元素样式慢慢的变化，常配合 hover 使用，增强网页交互体验
  • 属性名：translation
  • 常见取值：
    • 使用具体属性过渡时，格式：属性1 时长，属性2 时长，属性3 时长.....
  • 注意点：
    • 过度需要：默认状态 和 hover 状态样式不同，才有过渡效果
    • translation 属性经常给需要过渡的元素本身加（移入移除都有效果），并非加在 hover 伪类中
    • translation 属性设置在不同元素中，效果不同：
      • 给默认状态设置，鼠标移入移除都有过渡效果
      • 给 hover 状态设置，鼠标移入有过渡效果，移出没有过渡效果

  • 版心：网页的有效内容
  • margin：0 auto;

🌾 盒子阴影：

  • 作用：给盒子添加阴影效果，吸引用户注意，体现页面的制作细节
  • 属性名：box-shadow
  • 取值：

🌾 外边距折叠现象-① 合并现象：

  • 场景：垂直布局 的 块级元素，上下的 margin 会合并
  • 结果：最终两者距离为 margin 的最大值
  • 解决方法：避免两个盒子都有 margin
    • 只给其中一个盒子设置 margin

🌾 外边距折叠现象-② 塌陷现象：

  • 场景：互相嵌套 的 块级元素，子元素的 margin-top 会作用在父元素上
  • 结果：导致父元素一起向下移动
  • 解决方法：
    • 给父元素设置 border-top 或者 padding-top（分隔父子元素的 margin-top）

  - 给父元素设置over-flow：hidden
  - 转换成行内块元素
  - 设置浮动

🌾 行内元素的内外边距问题：

  • 如果想通过 margin 和 padding 改变行内元素的垂直位置，无法生效
  • 改变垂直位置的解决方法：
    • 通过 display 转换为行内块元素
    • 通过 line-height 设置行高改变垂直位置

✍️ 二、伪元素

  • 目标：能够使用伪元素在网页中创建内容
  • 伪元素：一般页面中的非主体内容可以使用伪元素
  • 区别：
    • 元素：HTML 设置的标签
    • 伪元素：由 CSS 模拟出的标签效果
  • 种类：
    • ::before：在父元素内容的最前添加一个伪元素
    • ::after：在父元素内容的最后添加一个伪元素
  • 注意点：
    • 必须设置 content 属性才生效
    • 伪元素默认是行内元素

✍️ 三、标准流（默认排版规则）

  • 常见标准流排版规则：
    • 块级元素：从上往下，垂直布局，独占一行
    • 行内元素 或 行内块元素：从左到右，水平布局，空间不够自动折行

✍️ 四、行内块问题

  • 行内元素和行内块元素在换行时会多出一个空格
  • 解决方法：
    • 要想排版清晰（换行），就不能转为行内块元素，使用浮动实现
    • 浮动过去是为了图片环绕，现在作用是网页布局

✍️ 五、浮动

  • 早期作用：图片环绕
  • 现在作用：网页布局
    • float：right/left实现 换行和不转换行内块元素 的情况下无间隔且在一行
  • 浮动特点：
    • 浮动元素会脱离标准流（脱标），在标准流中不占位置
      • 可以抽象为飘在空中，不再受控制
    • 浮动元素比标准流高半个级别，可以覆盖标准流中的元素
    • 浮动找浮动，下一个浮动元素会在上一个浮动元素后面左右浮动
    • 浮动的标签是顶对齐的（可以加 margin 调整位置）
    • 浮动元素有特殊的显示效果：
      • 一行可以显示多个
      • 可以设置宽高
    • 浮动后的标签具备行内块的特点（在一行排列，可以修改宽高）
  • 注意点：
    • 浮动的元素不能通过text-align:center;或margin:0 auto;居中
    • 可以通过外套父元素进行版心居中
      • 浮动的元素要么在左边，要么在右边

✍️ 六、CSS 属性书写顺序：浏览器执行效率高

  1. 浮动 / display
  2. 盒子模型相关属性：margin padding border 宽度 高度 背景色
  3. 文字样式

做网页主导航用 li 标签嵌套 a 标签，如果只用 a 标签，执行效率非常低

CSS 定位与装饰

✍️ 一、定位介绍

🌾 网页常见布局方式

        - 标准流
           - 块级元素独占一行 → 垂直布局
           - 行内元素/行内块元素一行显示多个 → 水平布局
        - 浮动
           - 可以让原本垂直布局的 块级元素变成水平布局
        - 定位
           - 可以让原本元素自由的摆放在网页的任意位置
           - 一般用于 盒子之间的层叠情况

🌾 定位常用应用场景

        - 可以解决盒子与盒子之间的层叠问题，可以层叠在其他盒子上面
           - 定位之后的元素层级更高，可以层叠在其他盒子上面
        - 可以让盒子始终固定在屏幕中的某个位置

✍️ 二、定位使用

🌾 定位的使用步骤

  • 设置定位方式 - 属性名：position - 常见属性值：

    ​ -

    • 设置偏移值
      • 偏移设置分为两个方向，水平和垂直方向各选一个使用
      • 选取的规则一般是就近原则
      • 🌾 相对定位

  • 自恋型定位，相对于自己之前的位置进行移动

  • 代码：position:relative;

  • 特点：

    • 需要配合方位属性实现移动
    • 相对于自己原来的位置进行移动
    • 在页面中占位置 → 没有脱离标准流控制（原来位置还是自己的）
    • 如果上top下buttom左left右right都有，以上top和左left为准

  • 应用场景：

    • 配合绝对定位组 CP
    • 用于小范围的移动（子绝父相）子标签绝对定位，父标签相对定位

🌾 绝对定位

  • 拼爹型定位，相对于非静态定位的父元素（广义，逐级往上）进行定位移动

  • 代码：position:absolute;
  • 特点：
    • 需要配合方位属性实现移动
    • 默认相对于浏览器可视区域进行移动
    • 在页面中不占位置 → 已经脱离标准流
    • 改变标签的显示模式，具备了行内块元素的特点
  • 应用场景：
    • 配合绝对定位组 CP（子绝父相）子标签绝对定位，父标签相对定位

🌾 使用绝对定位进行居中

  • 绝对定位的盒子不能用左右 margin auto 居中
  • 解决方法：
    • 偏移值使用百分比
    • 例如水平居中：
      • position:absolute;
      • left: 50%;
      • margin-left= 盒子宽度的一半;或者margin-right= 盒子宽度的一半;
      • 但有一种场景，后续要改盒子长度，偏移值也要改，甚至长度为奇数（CSS 手写不支持小数）
    • 垂直居中同理

🌾 使用位移进行居中

  • 使用 CSS3 的transform
  • 方法和上个类似：
    • 偏移值使用百分比
    • 例如水平居中：
      • position:absolute;
      • left: 50%;
      • transform: translate(-50% -50%);
      • 移动自身 宽度/高度 的一半，负值为向左/向上，正值为向右/向上

🌾 固定定位

  • 死心眼式定位，相对于浏览器进行定位移动
  • 代码：position: fixed;
  • 特点：
    • 需要配合方位属性实现移动
    • 相对于浏览器可视区域进行移动
    • 在页面中不占位置，已经脱离标准流控制
    • 具备行内块元素的特点
  • 应用场景：
    • 让盒子固定在屏幕中的某个位置

🌾 元素的层级关系

  • 不同布局方式的层级关系：
    • 标准流 < 浮动< 定位
  • 不同定位之间的层级关系：
    • 相对、绝对、固定默认层级相同
    • 此时 HTML 中写在最下面的元素层级更高，会覆盖上面的元素
    • z-index: 数值;数值越大，层级越高，默认值为 0（配合定位才生效）
